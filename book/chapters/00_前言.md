# Linux 应用开发大全

> 本文档面向嵌入式 / 服务器 Linux 的应用层开发，聚焦**用户态**编程。内核驱动另有专章，本书不展开。

## 为什么要有这份文档

Linux 应用开发涉及的接口散落在 POSIX、GNU libc、glibc 扩展、以及内核直接暴露的 syscall 里，初学者常见的痛点：

- 同一件事有好几套 API（比如 `open/read/write` 和 `fopen/fread/fwrite`），不知道什么时候用哪套
- 多线程、信号、IPC 的边界模糊，错用会导致偶发崩溃或数据错乱
- `man` 手册分 1/2/3 等多节，新手不知道自己该查哪节

这份文档按照"**一个主题一张速查表 + 可直接编译的最小例子 + 易错点**"的结构展开，目标是：遇到问题翻这份文档能比翻 man 快，查完立刻能动手。

## 使用约定

- 代码示例全部使用 **C99/C11**，编译器假定 GCC 9+
- 命令行提示符区分：`$` 普通用户，`#` root，`(target)$` 目标机（如开发板），`(host)$` 主机
- 系统调用后面标 `(2)`，库函数标 `(3)`，比如 `open(2)`、`printf(3)`，对应 `man 2 open`、`man 3 printf`
- 所有例子默认已包含 `#define _GNU_SOURCE` 并用 `-D_GNU_SOURCE` 编译，以启用扩展宏
