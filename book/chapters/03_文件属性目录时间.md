# 第 3 章 文件属性、目录与时间

## 3.1 为什么要理解文件元数据

Linux 里"一切皆文件"，但文件不只是一串字节，还有一套描述它的**元数据（metadata）**存在 inode 里。理解元数据是做以下事情的前提：

- 判断一个路径是否存在、是文件还是目录还是符号链接
- 按权限控制谁能读写执行
- 实现备份工具（检测文件是否被修改过）
- 遍历目录树（递归复制、查找、构建 Makefile 依赖）
- 正确处理符号链接（避免路径穿越安全漏洞）

## 3.2 stat / lstat / fstat

```c
#include <sys/stat.h>
int stat (const char *path, struct stat *buf);   /* 跟随符号链接 */
int lstat(const char *path, struct stat *buf);   /* 不跟随，返回链接本身信息 */
int fstat(int fd,           struct stat *buf);   /* 通过已打开的 fd */
int fstatat(int dirfd, const char *path, struct stat *buf, int flags);
```

### struct stat 的核心字段

```c
struct stat {
    dev_t     st_dev;     /* 设备号（所在文件系统） */
    ino_t     st_ino;     /* inode 号（同一文件系统内唯一） */
    mode_t    st_mode;    /* 文件类型 + 权限位 */
    nlink_t   st_nlink;   /* 硬链接数 */
    uid_t     st_uid;     /* 属主用户 ID */
    gid_t     st_gid;     /* 属组 ID */
    dev_t     st_rdev;    /* 设备文件的设备号（仅字符/块设备有意义） */
    off_t     st_size;    /* 文件字节大小（符号链接是目标路径长度） */
    blksize_t st_blksize; /* 文件系统建议的 IO 块大小（通常 4096） */
    blkcnt_t  st_blocks;  /* 分配的 512 字节块数（稀疏文件会 < st_size/512） */
    time_t    st_atime;   /* 最后访问时间（access） */
    time_t    st_mtime;   /* 最后内容修改时间（modify） */
    time_t    st_ctime;   /* 最后元数据变更时间（change，不是 create！） */
};
```

> **atime/mtime/ctime 三者的区别**  
> - `mtime`：文件内容变了（write）才更新，备份工具用它检测文件是否被修改  
> - `ctime`：权限、属主、链接数、mtime 本身变了都会更新  
> - `atime`：每次 read 都更新（开销大，所以现代内核挂载时默认用 `relatime`，只在 mtime/ctime 更新后才更新 atime）

### 调用链（以 stat 为例）

```
用户态 stat(path, buf)
  └─ syscall(SYS_newfstatat, AT_FDCWD, path, buf, 0)
       └─ vfs_statx()
            ├─ filename_lookup()       // 路径解析，得到 dentry
            ├─ vfs_getattr()
            │    └─ inode->i_op->getattr()  // 调用具体文件系统（ext4/xfs/...）
            │         └─ 从 inode 填充 kstat 结构
            └─ cp_statx_to_user()     // 把内核 kstat 复制到用户 buf
```

### 常用宏判断文件类型（st_mode）

```c
S_ISREG(m)   /* 普通文件 */
S_ISDIR(m)   /* 目录 */
S_ISLNK(m)   /* 符号链接（lstat 才能看到） */
S_ISCHR(m)   /* 字符设备 */
S_ISBLK(m)   /* 块设备 */
S_ISFIFO(m)  /* 命名管道 FIFO */
S_ISSOCK(m)  /* Unix 域 socket */
```

文件类型占 st_mode 的高 4 位，权限占低 12 位：

```
st_mode  = 0100644
           ├── 010  文件类型：普通文件（0010 = S_IFREG）
           └── 0644 权限：rw-r--r--
```

### 示例：递归统计目录下各类文件数量

```c
#define _GNU_SOURCE
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <dirent.h>
#include <sys/stat.h>
#include <unistd.h>

typedef struct { long reg, dir, lnk, other; } Counter;

void walk(const char *path, Counter *c)
{
    DIR *d = opendir(path);
    if (!d) return;

    struct dirent *ent;
    while ((ent = readdir(d)) != NULL) {
        if (!strcmp(ent->d_name, ".") || !strcmp(ent->d_name, "..")) continue;

        char full[4096];
        snprintf(full, sizeof(full), "%s/%s", path, ent->d_name);

        struct stat st;
        if (lstat(full, &st) < 0) continue;  /* lstat：不跟链接 */

        if      (S_ISREG(st.st_mode))  c->reg++;
        else if (S_ISDIR(st.st_mode)) { c->dir++; walk(full, c); }
        else if (S_ISLNK(st.st_mode))  c->lnk++;
        else                           c->other++;
    }
    closedir(d);
}

int main(int argc, char *argv[])
{
    const char *root = argc > 1 ? argv[1] : ".";
    Counter c = {0};
    walk(root, &c);
    printf("普通文件: %ld\n目录: %ld\n符号链接: %ld\n其他: %ld\n",
           c.reg, c.dir, c.lnk, c.other);
    return 0;
}
```

## 3.3 权限位与 umask

### 权限位含义

| 权限位 | 八进制 | 对普通文件含义 | 对目录含义 |
| --- | --- | --- | --- |
| `S_IRUSR` | 0400 | 属主可读 | 属主可列目录 |
| `S_IWUSR` | 0200 | 属主可写 | 属主可在目录内创建/删除文件 |
| `S_IXUSR` | 0100 | 属主可执行 | 属主可进入（cd）目录 |
| `S_IRGRP` | 0040 | 属组可读 | 同上，属组 |
| `S_IWGRP` | 0020 | 属组可写 | 同上 |
| `S_IXGRP` | 0010 | 属组可执行 | 同上 |
| `S_IROTH` | 0004 | 其他可读 | 同上 |
| `S_IWOTH` | 0002 | 其他可写 | 同上 |
| `S_IXOTH` | 0001 | 其他可执行 | 同上 |
| `S_ISUID` | 04000 | SetUID：执行时以属主身份运行 | 忽略 |
| `S_ISGID` | 02000 | SetGID：执行时以属组身份运行 | 新文件继承目录属组 |
| `S_ISVTX` | 01000 | 粘着位（现已无效） | 只有文件属主才能删除目录内的文件 |

### umask —— 创建文件时的权限掩码

`open(path, flags, 0666)` 里的 `0666` 不是最终权限，而是**期望权限**。内核会用以下公式计算实际权限：

```
实际权限 = 期望权限 & ~umask
```

比如 umask = 022（常见默认值），则 `open(..., 0666)` 得到 `0666 & ~022 = 0644`。

```c
#include <sys/stat.h>
mode_t umask(mode_t mask);  /* 设置并返回旧 umask，永远成功 */
```

> umask 是**进程级别**的，Shell 里设置 `umask 027` 只影响当前 Shell 及其子进程。

### chmod / fchmod / chown / fchown

```c
int chmod (const char *path, mode_t mode);
int fchmod(int fd, mode_t mode);
int chown (const char *path, uid_t owner, gid_t group);
int fchown(int fd, uid_t owner, gid_t group);
```

只有文件属主或 root 能 `chmod`；只有 root 能把文件转让给别人（`chown`）。

## 3.4 目录操作

### 创建与删除

```c
#include <sys/stat.h>
#include <unistd.h>
int mkdir (const char *path, mode_t mode);  /* 创建，mode 受 umask 影响 */
int rmdir (const char *path);               /* 只能删空目录 */
int rename(const char *old, const char *new); /* 移动/重命名，同一文件系统内是原子操作 */
```

`rename` 是原子的原因：内核只需修改目录项（dentry）的指向，不移动实际数据，整个操作在一个 VFS 事务里完成。这是"先写临时文件再 rename 替换"技巧的基础——可以保证即使过程中崩溃，目标文件要么是新的完整版，要么是旧版，不会出现中间状态。

### 遍历目录：opendir / readdir / closedir

```c
#include <dirent.h>
DIR           *opendir (const char *name);
DIR           *fdopendir(int fd);          /* 从已打开的 dirfd 建流 */
struct dirent *readdir (DIR *dirp);        /* 线程不安全；readdir_r 已废弃，不用 */
int            closedir(DIR *dirp);
void           rewinddir(DIR *dirp);
```

`struct dirent` 的关键字段：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `d_ino` | `ino_t` | inode 号 |
| `d_type` | `unsigned char` | 文件类型（`DT_REG` / `DT_DIR` / `DT_LNK` 等，可能是 `DT_UNKNOWN` 需 stat 确认） |
| `d_name` | `char[]` | 文件名（不含路径），最长 255 字节 + `\0` |

> `d_type` 并非所有文件系统都支持（XFS 老版本、某些网络文件系统会返回 `DT_UNKNOWN`），需要 `stat` 确认类型时保底。

### 为什么 readdir 是线程不安全的

`readdir` 返回的是指向 DIR 内部静态缓冲的指针，同一 `DIR*` 在多线程里并发调用会相互覆盖。正确做法是每个线程各自 `opendir`，或者用 `readdir` 读完立刻拷贝 `d_name`。

### nftw —— 递归目录遍历的正确工具

手写递归 `opendir/readdir` 有很多边界要处理（符号链接循环、过深递归、跨设备）。GNU 提供了更安全的 `nftw`：

```c
#define _XOPEN_SOURCE 700
#include <ftw.h>
int nftw(const char *dirpath,
         int (*fn)(const char *fpath, const struct stat *sb,
                   int typeflag, struct FTW *ftwbuf),
         int nopenfd,   /* 同时打开的最大目录数，推荐 16 */
         int flags);    /* FTW_PHYS（不跟符号链接）FTW_DEPTH（先子后父）等 */
```

`typeflag` 常见值：`FTW_F`（普通文件）、`FTW_D`（目录进入前）、`FTW_DP`（目录遍历完后，需 `FTW_DEPTH`）、`FTW_SL`（符号链接）。

```c
static int print_file(const char *path, const struct stat *sb,
                       int type, struct FTW *ftwb)
{
    if (type == FTW_F)
        printf("%s  (%lld bytes)\n", path, (long long)sb->st_size);
    return 0;
}

nftw("/etc", print_file, 16, FTW_PHYS);
```

### 获取当前工作目录

```c
#include <unistd.h>
char *getcwd(char *buf, size_t size);   /* 写到 buf，失败返回 NULL */
char *get_current_dir_name(void);       /* GNU 扩展，自动 malloc，用完要 free */
int   chdir(const char *path);          /* 改变当前进程的工作目录 */
int   fchdir(int fd);                   /* 通过 dirfd 切换，线程安全 */
```

## 3.5 硬链接与符号链接

### 硬链接（hard link）

```c
#include <unistd.h>
int link(const char *oldpath, const char *newpath);    /* 创建硬链接 */
int linkat(int olddirfd, const char *oldpath,
           int newdirfd, const char *newpath, int flags);
int unlink(const char *path);   /* 删除目录项，当 nlink==0 且无进程打开时才真正删文件 */
```

**本质**：硬链接是同一 inode 的多个目录项。`stat` 里的 `st_nlink` 就是硬链接数。

- 同一文件系统内才能创建（不同设备间不行）
- 不能对目录创建硬链接（会破坏目录树的 DAG 结构，`..` 会乱）
- 删除文件本质是 unlink：只要还有其他硬链接或进程打开它，数据就不会丢

`unlink` 的延迟删除特性很有用：打开文件后立刻 unlink，文件从目录里消失（别人看不到），但进程里的 fd 仍然有效，进程退出后才真正删除。这是实现"自动清理的临时文件"的标准做法：

```c
int fd = open("/tmp/tmpXXX", O_RDWR | O_CREAT | O_EXCL, 0600);
unlink("/tmp/tmpXXX");  /* 目录项消失，但 fd 还能用，进程退出自动清理 */
```

### 符号链接（symbolic link / symlink）

```c
int symlink(const char *target, const char *linkpath);
ssize_t readlink(const char *path, char *buf, size_t bufsiz);  /* 读取链接目标 */
int readlinkat(int dirfd, const char *path, char *buf, size_t bufsiz);
```

**本质**：符号链接是一个特殊文件，内容是目标路径字符串。

| 特性 | 硬链接 | 符号链接 |
| --- | --- | --- |
| 跨文件系统 | 不支持 | 支持 |
| 指向目录 | 不支持 | 支持 |
| 目标被删后 | 仍然有效 | 变成悬空链接 |
| stat 看到的 | 目标 inode | 链接本身（lstat）或目标（stat） |
| 删除链接 | unlink（减 nlink） | unlink（删链接文件本身） |

> **安全警告**：`stat` 会跟随符号链接，在处理用户提供的路径时一定要用 `lstat` + 检查 `S_ISLNK`，防止路径穿越攻击（如 `/tmp/link -> /etc/passwd`）。

## 3.6 时间操作

### Linux 的时钟类型

Linux 提供多个时钟，通过 `clock_gettime(clk_id, &ts)` 获取：

| 时钟 ID | 含义 | 单调性 | 典型用途 |
| --- | --- | --- | --- |
| `CLOCK_REALTIME` | 墙钟时间（UTC 秒数） | 否（NTP 可调） | 日志时间戳、文件时间 |
| `CLOCK_MONOTONIC` | 单调时钟（系统启动后不回退） | 是 | 测量耗时、定时器 |
| `CLOCK_MONOTONIC_RAW` | 不受 NTP 频率调整的单调时钟 | 是 | 高精度性能测量 |
| `CLOCK_PROCESS_CPUTIME_ID` | 本进程消耗的 CPU 时间 | 是 | 性能剖析 |
| `CLOCK_THREAD_CPUTIME_ID` | 本线程消耗的 CPU 时间 | 是 | 线程级性能剖析 |
| `CLOCK_BOOTTIME` | 包含睡眠时间的单调时钟 | 是 | 计算设备运行时长 |

**为什么测耗时一定要用 `CLOCK_MONOTONIC` 而不是 `time()`？**  
`time()` 返回 `CLOCK_REALTIME`，NTP 同步时可能突然向前或向后跳。用它计算时间差，如果中间发生了 NTP 调整，结果会是负数或虚假的大值。

```c
#include <time.h>
struct timespec t0, t1;
clock_gettime(CLOCK_MONOTONIC, &t0);
/* ... 要测量的代码 ... */
clock_gettime(CLOCK_MONOTONIC, &t1);
double ms = (t1.tv_sec - t0.tv_sec) * 1e3
          + (t1.tv_nsec - t0.tv_nsec) / 1e6;
printf("耗时: %.3f ms\n", ms);
```

### 时间格式转换

```c
#include <time.h>
time_t    time(NULL);                      /* 返回 Unix 时间戳（秒） */
struct tm *localtime_r(const time_t *, struct tm *);  /* 线程安全版 */
struct tm *gmtime_r   (const time_t *, struct tm *);  /* UTC，线程安全 */
time_t     mktime(struct tm *);            /* struct tm -> time_t（本地时区） */
char      *strftime(char *, size_t, const char *fmt, const struct tm *); /* 格式化 */
int        strptime(const char *, const char *fmt, struct tm *);         /* 解析 */
```

常用格式串：

| 格式 | 含义 | 示例输出 |
| --- | --- | --- |
| `%Y-%m-%d %H:%M:%S` | 标准日期时间 | `2026-04-23 15:30:00` |
| `%a, %d %b %Y` | HTTP 日期格式 | `Thu, 23 Apr 2026` |
| `%s` | Unix 时间戳字符串（glibc 扩展） | `1745000000` |
| `%Z` | 时区缩写 | `CST` |

### 修改文件时间戳

```c
#include <sys/stat.h>
#include <fcntl.h>
/* 现代接口，纳秒精度 */
int utimensat(int dirfd, const char *path,
              const struct timespec times[2], int flags);
int futimens(int fd, const struct timespec times[2]);
/* times[0] = atime, times[1] = mtime
   tv_nsec 设为 UTIME_NOW  表示设为当前时间
   tv_nsec 设为 UTIME_OMIT 表示不改这个时间戳 */
```

> 旧版 `utime()`/`utimes()` 只有秒/微秒精度，已不推荐。

### 高精度定时器 timerfd

`timerfd` 是 Linux 特有的接口，把定时器封装成一个 fd，可以用 `epoll` 统一监听，非常适合事件循环：

```c
#include <sys/timerfd.h>
int timerfd_create(int clockid, int flags);  /* flags: TFD_NONBLOCK | TFD_CLOEXEC */
int timerfd_settime(int fd, int flags,
                    const struct itimerspec *new_value,
                    struct itimerspec *old_value);
```

```c
int tfd = timerfd_create(CLOCK_MONOTONIC, TFD_NONBLOCK | TFD_CLOEXEC);

struct itimerspec its = {
    .it_interval = { .tv_sec = 1, .tv_nsec = 0 },  /* 每 1 秒触发 */
    .it_value    = { .tv_sec = 1, .tv_nsec = 0 },  /* 首次延迟 1 秒 */
};
timerfd_settime(tfd, 0, &its, NULL);

/* 用 epoll 监听 tfd，定时到期时 read 出触发次数 */
uint64_t expirations;
read(tfd, &expirations, sizeof(expirations));  /* 阻塞或 epoll 通知后调用 */
printf("触发了 %llu 次\n", (unsigned long long)expirations);
```

## 3.7 access / faccessat —— 权限检查（及其陷阱）

```c
#include <unistd.h>
int access  (const char *path, int mode);
int faccessat(int dirfd, const char *path, int mode, int flags);
/* mode: F_OK（是否存在）| R_OK | W_OK | X_OK */
```

**access 检查的是进程的实际 UID/GID**（不是有效 UID），SetUID 程序用 access 检查的是调用者的真实权限。

### 为什么不推荐用 access 做"先检查后操作"

```c
if (access("file", W_OK) == 0)  /* ← 此刻文件存在且可写 */
    fd = open("file", O_WRONLY); /* ← 此刻可能已被删除或替换（TOCTTOU 竞态） */
```

**正确做法**：直接 `open`，检查返回值的 `errno`，而不是提前 access。或者用 `openat` + `O_NOFOLLOW` 防符号链接替换。

## 3.9 inotify —— 文件系统事件监听

`inotify` 是 Linux 内核提供的文件/目录变更通知机制，热加载配置、构建系统（如 Watchman）都靠它：

```c
#include <sys/inotify.h>
int inotify_init1(int flags);             /* 返回 inotify fd，flags: IN_NONBLOCK | IN_CLOEXEC */
int inotify_add_watch(int fd, const char *path, uint32_t mask);
int inotify_rm_watch(int fd, int wd);
```

常用 mask 事件：

| mask | 触发条件 |
| --- | --- |
| `IN_CREATE` | 目录下有文件/子目录被创建 |
| `IN_DELETE` | 目录下有文件/子目录被删除 |
| `IN_MODIFY` | 文件内容被修改（write/truncate） |
| `IN_CLOSE_WRITE` | 以写方式打开的文件被关闭（更可靠的"修改完成"信号） |
| `IN_MOVED_FROM` / `IN_MOVED_TO` | rename 的源/目标（配合 cookie 可配对） |
| `IN_ATTRIB` | 权限、时间戳等元数据变更 |

读出的事件格式：

```c
struct inotify_event {
    int     wd;       /* watch 描述符 */
    uint32_t mask;    /* 事件类型 */
    uint32_t cookie;  /* 用于配对 MOVED_FROM/TO */
    uint32_t len;     /* name 字段长度（含对齐填充） */
    char    name[];   /* 触发事件的文件名（目录事件才有） */
};
```

```c
int ifd = inotify_init1(IN_NONBLOCK | IN_CLOEXEC);
int wd  = inotify_add_watch(ifd, "/etc", IN_MODIFY | IN_CREATE | IN_DELETE);

char buf[4096] __attribute__((aligned(__alignof__(struct inotify_event))));
for (;;) {
    ssize_t n = read(ifd, buf, sizeof(buf));
    for (char *p = buf; p < buf + n; ) {
        struct inotify_event *ev = (struct inotify_event *)p;
        if (ev->mask & IN_MODIFY)
            printf("修改: %s\n", ev->len ? ev->name : "(文件本身)");
        p += sizeof(*ev) + ev->len;
    }
}
```

## 3.10 易错点汇总

- **混淆 mtime 和 ctime**：ctime 不是创建时间（creation time），Linux 没有原生的创建时间（`statx` 新增了 `stx_btime` 但文件系统不一定记录）
- **stat 跟随符号链接**：处理用户输入路径时用 lstat，防止"stat 说是普通文件但实际是链接指向 /dev/mem"
- **readdir 返回的指针只保证到下一次 readdir 之前有效**：需要保存文件名时立刻 `strdup(ent->d_name)`
- **mkdir 的 mode 受 umask 影响**：想要 0777 实际可能得到 0755，创建后用 chmod 显式设置
- **rename 跨文件系统失败**：返回 `EXDEV`，需要 copy + unlink 代替
- **unlink 正在被进程打开的文件**：文件从目录树消失，但数据不删除，进程退出后才真正回收——是特性不是 bug
- **inotify 监听目录只能看到一层**：不递归，需要对每个子目录也 add_watch
- **time() 精度只有秒**：性能测量用 clock_gettime(CLOCK_MONOTONIC)，日志时间戳用 clock_gettime(CLOCK_REALTIME)
