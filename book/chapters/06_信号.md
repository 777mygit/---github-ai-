# 第 6 章 信号

## 6.1 信号是什么

信号是 Linux 内核向进程传递**异步事件通知**的机制。和 IPC 不同，信号不携带数据（只有信号编号），更像一个"中断"：进程随时可能收到，必须提前注册处理器。

信号的来源：

- **内核**：进程访问非法地址（SIGSEGV）、整除零（SIGFPE）、执行了非法指令（SIGILL）
- **终端**：`Ctrl+C`（SIGINT）、`Ctrl+\`（SIGQUIT）、`Ctrl+Z`（SIGTSTP）
- **其他进程**：`kill(pid, sig)`、`tkill(tid, sig)`
- **计时器**：`alarm()`、`timer_create()`
- **子进程退出**：SIGCHLD

## 6.2 常用信号速查

| 信号 | 编号 | 默认动作 | 典型来源 |
| --- | --- | --- | --- |
| `SIGHUP` | 1 | 终止 | 控制终端断开，也用于通知守护进程重新加载配置 |
| `SIGINT` | 2 | 终止 | `Ctrl+C` |
| `SIGQUIT` | 3 | 终止+core | `Ctrl+\` |
| `SIGILL` | 4 | 终止+core | 非法指令 |
| `SIGTRAP` | 5 | 终止+core | 调试断点、ptrace |
| `SIGABRT` | 6 | 终止+core | `abort()` 调用 |
| `SIGBUS` | 7 | 终止+core | 内存对齐错误、mmap 访问越界 |
| `SIGFPE` | 8 | 终止+core | 浮点/整数运算异常 |
| `SIGKILL` | 9 | 终止 | **不可捕获、不可忽略** |
| `SIGSEGV` | 11 | 终止+core | 非法内存访问 |
| `SIGPIPE` | 13 | 终止 | 写入已关闭读端的管道/socket |
| `SIGALRM` | 14 | 终止 | `alarm()` 超时 |
| `SIGTERM` | 15 | 终止 | `kill` 命令默认发送，可捕获 |
| `SIGCHLD` | 17 | 忽略 | 子进程状态改变 |
| `SIGCONT` | 18 | 继续 | 让被停止的进程继续运行 |
| `SIGSTOP` | 19 | 停止 | **不可捕获、不可忽略** |
| `SIGTSTP` | 20 | 停止 | `Ctrl+Z` |
| `SIGUSR1` | 10 | 终止 | 用户自定义，程序自己决定含义 |
| `SIGUSR2` | 12 | 终止 | 用户自定义 |
| `SIGWINCH` | 28 | 忽略 | 终端窗口大小变化 |
| `SIGRTMIN~SIGRTMAX` | 34~64 | 终止 | 实时信号，可排队，带附加数据 |

## 6.3 信号的生命周期

```
信号产生（generate）
   │
   ▼
内核把信号标记为"待处理"（pending）
在目标进程的 task_struct->pending（或 signal->shared_pending）中置位
   │
   ▼
目标进程下次从内核态返回用户态时，检查 pending 信号
   │
   ├── 信号被阻塞（sigmask 置位）→ 继续等待，不递达
   │
   ▼
信号递达（deliver）
   ├── 默认动作（SIG_DFL）：终止、core dump、停止、忽略
   ├── 忽略（SIG_IGN）
   └── 用户处理器（signal handler）
          │
          ▼
       保存当前寄存器状态到用户栈（sigframe）
       跳转到处理器函数执行
       处理器返回后恢复寄存器，继续被中断的代码
```

**实时信号（SIGRTMIN~SIGRTMAX）与普通信号的区别**：

| 特性 | 普通信号（1~31） | 实时信号（34~64） |
| --- | --- | --- |
| 多个相同信号排队 | 合并为 1 个（可能丢失） | 严格排队，不丢失 |
| 带附加数据 | 否 | 是（siginfo_t 里的 si_value） |
| 发送顺序 | 不保证 | 低编号先递达 |

## 6.4 注册信号处理器

### signal()（简单但有坑）

```c
#include <signal.h>
typedef void (*sighandler_t)(int);
sighandler_t signal(int signum, sighandler_t handler);
/* handler: SIG_DFL（恢复默认）| SIG_IGN（忽略）| 函数指针 */
```

`signal` 的行为在 POSIX 和不同 Unix 实现里不一致（主要是处理器调用后是否自动重置），**不推荐在新代码里使用**。

### sigaction()（推荐）

```c
#include <signal.h>
int sigaction(int signum, const struct sigaction *act, struct sigaction *oldact);

struct sigaction {
    void     (*sa_handler)(int);              /* 简单处理器 */
    void     (*sa_sigaction)(int, siginfo_t *, void *);  /* 带信息的处理器（SA_SIGINFO 时用） */
    sigset_t   sa_mask;     /* 处理器执行期间额外阻塞的信号 */
    int        sa_flags;    /* 见下表 */
};
```

| sa_flags | 含义 |
| --- | --- |
| `SA_RESTART` | 被信号打断的系统调用自动重启（强烈推荐） |
| `SA_SIGINFO` | 使用 sa_sigaction 三参数版本，获取 siginfo_t |
| `SA_NOCLDWAIT` | SIGCHLD 时不产生僵尸（等同 SIG_IGN 但不忽略信号） |
| `SA_NOCLDSTOP` | SIGCHLD 只在子进程退出时触发，停止不触发 |
| `SA_ONESHOT` | 处理器调用一次后重置为 SIG_DFL |
| `SA_NODEFER` | 处理器执行期间不阻塞当前信号自身 |
| `SA_RESETHAND` | 同 SA_ONESHOT |

### 标准 sigaction 模板

```c
#define _GNU_SOURCE
#include <stdio.h>
#include <stdlib.h>
#include <signal.h>
#include <string.h>
#include <unistd.h>
#include <stdatomic.h>

static volatile sig_atomic_t g_stop = 0;  /* 信号处理器与主循环共享的标志 */

static void on_term(int sig)
{
    /* 信号处理器里只能调用"异步信号安全"函数！
       write() 是安全的，printf() 不是 */
    const char msg[] = "收到终止信号，准备退出\n";
    write(STDOUT_FILENO, msg, sizeof(msg) - 1);
    g_stop = 1;
}

int main(void)
{
    struct sigaction sa = {
        .sa_handler = on_term,
        .sa_flags   = SA_RESTART,
    };
    sigemptyset(&sa.sa_mask);

    sigaction(SIGTERM, &sa, NULL);
    sigaction(SIGINT,  &sa, NULL);

    while (!g_stop) {
        /* 主循环 */
        sleep(1);
        printf("running...\n");
        fflush(stdout);
    }

    printf("退出\n");
    return 0;
}
```

## 6.5 异步信号安全（Async-Signal Safety）

信号处理器可能在**任意时刻**打断主程序，包括正在执行 `malloc`、`printf` 的中间。这些函数内部有锁或全局状态，如果信号处理器也调用它们，会导致死锁或数据损坏。

**只有 POSIX 明确列为"异步信号安全"的函数才能在信号处理器里调用**，包括：

- `_exit`、`abort`（但 abort 发 SIGABRT）
- `write`、`read`、`close`、`open`、`send`、`recv`
- `kill`、`raise`、`sigprocmask`
- `sem_post`（POSIX）
- `clock_gettime`、`getpid`、`getppid`

**不安全的函数**（不能在信号处理器里调用）：

- `printf`、`fprintf`、`sprintf`（有内部锁）
- `malloc`、`free`（堆锁）
- `exit`（冲刷 stdio）
- `strerror`（静态缓冲区）
- C++ 析构函数和异常

### 最佳实践：信号处理器只设标志，主循环检查

```c
static volatile sig_atomic_t g_reload = 0;

static void on_hup(int sig) { g_reload = 1; }  /* 只做这一件事 */

/* 主循环 */
while (1) {
    if (g_reload) {
        g_reload = 0;
        reload_config();   /* 在主循环里安全地做复杂操作 */
    }
    do_work();
}
```

## 6.6 信号集与信号屏蔽

```c
#include <signal.h>
int sigemptyset(sigset_t *set);       /* 清空 */
int sigfillset (sigset_t *set);       /* 全置 */
int sigaddset  (sigset_t *set, int sig);
int sigdelset  (sigset_t *set, int sig);
int sigismember(const sigset_t *set, int sig);

int sigprocmask(int how, const sigset_t *set, sigset_t *oldset);
/* how: SIG_BLOCK | SIG_UNBLOCK | SIG_SETMASK */

int sigpending(sigset_t *set);   /* 查询哪些信号正在等待递达（被阻塞的 pending） */
```

**为什么需要信号屏蔽？**

场景：主线程正在操作一个链表，此时 SIGTERM 处理器里也操作同一链表，会导致数据结构损坏。正确做法：进入临界区前屏蔽信号，出临界区后恢复。

```c
sigset_t block_all, old;
sigfillset(&block_all);
sigprocmask(SIG_BLOCK, &block_all, &old);   /* 进临界区：屏蔽所有信号 */
/* ... 操作共享数据 ... */
sigprocmask(SIG_SETMASK, &old, NULL);       /* 出临界区：恢复 */
```

## 6.7 sigsuspend 和 sigwaitinfo —— 等待信号

### sigsuspend：原子地"解除屏蔽 + 睡眠等待"

```c
int sigsuspend(const sigset_t *mask);
/* 用 mask 替换当前 sigmask，然后挂起，直到有非 mask 阻塞的信号递达
   信号处理器返回后，恢复原来的 sigmask，sigsuspend 返回 -1 EINTR */
```

这解决了"先解除屏蔽，再 sleep"之间的竞态：如果信号在解除屏蔽和 sleep 之间到来，sleep 会错过它。`sigsuspend` 把这两步合并为原子操作。

### sigwaitinfo / sigtimedwait：把信号同步化

```c
int sigwaitinfo(const sigset_t *set, siginfo_t *info);
int sigtimedwait(const sigset_t *set, siginfo_t *info,
                 const struct timespec *timeout);
```

先用 `sigprocmask` 屏蔽想等待的信号，然后在主循环里用 `sigwaitinfo` 同步等待——信号被同步处理，完全不用异步信号处理器：

```c
sigset_t wait_set;
sigemptyset(&wait_set);
sigaddset(&wait_set, SIGTERM);
sigaddset(&wait_set, SIGINT);
sigaddset(&wait_set, SIGHUP);
sigprocmask(SIG_BLOCK, &wait_set, NULL);  /* 屏蔽，让它们进 pending 队列 */

for (;;) {
    siginfo_t info;
    int sig = sigwaitinfo(&wait_set, &info);  /* 同步等待 */
    if (sig == SIGTERM || sig == SIGINT) break;
    if (sig == SIGHUP) reload_config();
}
```

## 6.8 signalfd —— 把信号变成 fd，用 epoll 统一处理

```c
#include <sys/signalfd.h>
int signalfd(int fd, const sigset_t *mask, int flags);
/* fd = -1 创建新的，否则更新已有的 signalfd
   flags: SFD_NONBLOCK | SFD_CLOEXEC */
```

`signalfd` 让你用 `read` 读出信号，`epoll` 监听 signalfd，和其他 IO 事件统一放在一个事件循环里处理。这是现代事件驱动程序的推荐做法。

```c
sigset_t mask;
sigemptyset(&mask);
sigaddset(&mask, SIGTERM);
sigaddset(&mask, SIGINT);
sigprocmask(SIG_BLOCK, &mask, NULL);  /* 先屏蔽，防止默认处理 */

int sfd = signalfd(-1, &mask, SFD_NONBLOCK | SFD_CLOEXEC);

/* 把 sfd 加入 epoll，有事件时 read */
struct signalfd_siginfo info;
read(sfd, &info, sizeof(info));
printf("收到信号 %u，来自 PID %u\n", info.ssi_signo, info.ssi_pid);
```

## 6.9 发送信号

```c
#include <signal.h>
int kill   (pid_t pid, int sig);   /* 发给进程（pid<0 发给进程组） */
int killpg (pid_t pgrp, int sig);  /* 发给进程组 */
int raise  (int sig);              /* 发给自己（等同 kill(getpid(), sig)） */
int tgkill (pid_t tgid, pid_t tid, int sig);  /* 发给指定线程 */
```

发 `sig=0` 是一个**探测技巧**：不真正发信号，只检查 pid 是否存在和是否有权限发信号。

```c
if (kill(pid, 0) == 0)
    printf("进程 %d 存在\n", pid);
else if (errno == ESRCH)
    printf("进程 %d 不存在\n", pid);
else if (errno == EPERM)
    printf("无权限\n");
```

### alarm 和 setitimer

```c
unsigned int alarm(unsigned int seconds);   /* seconds 后发 SIGALRM，返回上次剩余秒数 */

#include <sys/time.h>
int setitimer(int which, const struct itimerval *new, struct itimerval *old);
/* which: ITIMER_REAL（发 SIGALRM）| ITIMER_VIRTUAL（进程 CPU 时间）| ITIMER_PROF */
```

`alarm` 只有秒精度，`setitimer` 有微秒精度。更现代的做法是 `timer_create`（POSIX）配合 `timerfd_create`（Linux 特有，可 epoll）。

## 6.10 信号与多线程

多线程下，信号的行为更复杂：

- **进程级信号**（如 SIGTERM）：内核选**任意一个**没有屏蔽该信号的线程来递达
- **线程级信号**（`tgkill`）：发给指定线程
- **每个线程都有自己的 `sigmask`**，可以用 `pthread_sigmask` 单独设置

**标准做法**：在 `main` 线程 `pthread_create` 之前屏蔽所有信号，这样所有子线程继承屏蔽。然后单独创建一个"信号处理线程"用 `sigwaitinfo` 同步等待：

```c
#include <stdio.h>
#include <stdlib.h>
#include <signal.h>
#include <pthread.h>
#include <unistd.h>

static sigset_t g_sigset;

static void *sig_thread(void *arg)
{
    for (;;) {
        int sig;
        sigwait(&g_sigset, &sig);
        printf("信号线程收到: %d\n", sig);
        if (sig == SIGTERM || sig == SIGINT) {
            exit(0);
        }
    }
    return NULL;
}

int main(void)
{
    sigemptyset(&g_sigset);
    sigaddset(&g_sigset, SIGTERM);
    sigaddset(&g_sigset, SIGINT);
    sigaddset(&g_sigset, SIGHUP);
    pthread_sigmask(SIG_BLOCK, &g_sigset, NULL); /* 主线程（及所有后续子线程）屏蔽 */

    pthread_t tid;
    pthread_create(&tid, NULL, sig_thread, NULL);
    pthread_detach(tid);

    /* 其他工作线程 */
    pause();
    return 0;
}
```

## 6.11 SIGCHLD —— 子进程状态变化

`SIGCHLD` 在子进程**退出、被停止、被继续**时由内核发给父进程。注册 SIGCHLD 处理器来异步回收子进程：

```c
static void on_child(int sig)
{
    /* 循环 wait，因为多个子进程同时退出时信号可能只收到一次 */
    int status;
    pid_t pid;
    while ((pid = waitpid(-1, &status, WNOHANG)) > 0) {
        /* 处理每个退出的子进程 */
    }
}

struct sigaction sa = { .sa_handler = on_child, .sa_flags = SA_RESTART | SA_NOCLDSTOP };
sigemptyset(&sa.sa_mask);
sigaction(SIGCHLD, &sa, NULL);
```

> **为什么要循环 waitpid 而不是只 wait 一次？**  
> 如果 3 个子进程同时退出，内核可能只发了 1 次 SIGCHLD（信号不排队），一次 wait 只回收 1 个，另外 2 个成为僵尸。

## 6.12 易错点汇总

- **信号处理器里调用 printf / malloc**：不是异步信号安全函数，会导致死锁或堆损坏
- **用普通 int 而非 sig_atomic_t 做标志**：多核上不保证对 int 的写是原子的，可能被优化掉或读到旧值
- **SIGPIPE 没处理就写 socket**：进程被直接 kill，写 TCP 后对端关闭时常见
- **多线程里 signal() 设置处理器**：signal() 不是线程安全的，用 sigaction()
- **SIGCHLD 只 wait 一次**：多子进程同时退出只收到一次信号，必须循环 waitpid(WNOHANG)
- **忽略 SIGPIPE 没考虑返回值**：`signal(SIGPIPE, SIG_IGN)` 后，write 返回 -1 EPIPE，要检查返回值
- **实时信号发送者过快**：接收方处理不过来，内核 pending 队列（默认 SIGQUEUE_MAX=32768）满后发送失败 EAGAIN
